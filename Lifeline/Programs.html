    # 项目一、保时捷APP小程序微信移植（服务器到期，已下线）

    # 项目描述(2023.2.10~2023.3.20)

    该项目将保时捷官方app的界面和功能移植到微信小程序上，实现了简易的用户单点登陆、注册、验证功能，以及用户数据的存储，将 各个功能页和交互通过微信原生实现，后端则是基于nodejs，使用mysql+redis+eggjs框架进行部署，独立开发。

    # 项目详情

    1. 保时捷官方app使用起来响应比较慢，用户体验较差，将其移植到微信小程序上，登陆更方便且交互响应更及时。本项目前端部分共设 计了两大模块，一个是用户登陆注册模块,还有一个是进入系统后的用户界面模块。

    2. 进入用车系统后的页面默认是“状态0”的车型页面，用户可以通过顶部的切换车辆按钮来切换不同的车型页面，所有页面基本做到了对 官方app1:1的还原。为了提高代码的可读性，并且在页面设计上做到更高的简洁度和清晰度，本人将众多功能封装成组件来使用。

    3. 本项目有使用到较多阿里云产品，如oss服务，sms服务，以及ecs服务。后台是通过docker将项目部署在阿里云服务器上，基 于egg.js，通过redis+sms服务完成验证，再通过mysql对用户数据进行存储和验证，前端则是通过微信原生开发框架进行编写。

    # 项目二、车百科微信小程序（仿小红书【图+文】，服务器到期，已下线）

    # 项目描述(2023.3.22~2023.4.23)

    该项目是仿“小红书app”页面结构的一个app，实现用图片+文字的形势来介绍各种品牌的车的文字资料和图片资料，独立开发。

    # 项目详情

    1．项目架构：前端框架采用的是taro框架（车百科app）和ant-design-pro框架（车百科app后台管理页面），后端框架使 用的是egg.js框架，整体项目分为管理端和用户端。

    2．用户在微信上搜索“车百科”进入到默认页面，可以通过tab按钮来筛选车型。管理员可以登陆到后台管 理页面，为车百科添加物品数据，也可以增加新的管理员用户，用户无需注册可以直接访问app并使用app的功能。

    # 项目三、Ai1000项目（管理端和用户端开发）

    # 项目地址：Ai1000 (opens new window)

    # 项目描述(2023.7~2023.9)

    本项目是在开源框架基础上进行二开的（旧版允许sass部署）为电商老板进行sass定制的一套人工智能商务咨询系统，集成了GPT对话、知识库、小应用模型发布与管理、分销系统等核心功能，贡献代码1w+。

    # 项目详情

    1. 项目架构：整个项目的用户端使用了Next框架，框架UI选择Chakra-ui+tailwindcss，数据库使用了mongodb+postsql。管理端则是使用Tushan框架进行快速构建。
    2. 项目二开功能：
    • 用户的登陆注册通过next的api实现，注册时短信验证接了第三方短信平台联麓的api，通过CryptoJS+md5的方式加密signatrue以保证用户信息安全性。
    • 用户的充值和消费对接了微信支付的api，在组长的带领下理解并实现了用户的微信支付功能。
    • 小模型应用市场，对于每个训练好的应用小模型（提示词优化/知识库模型训练后）管理员可以将它们分享到应用市场，供其他用户使用。
    • 用户通过充值指定产品可以给上级和上上级邀请人进行反佣，不同等级的用户返佣比例也不一致，
      获取的佣金可以提现。
    • 小应用模型可修改公告，可查看累计消耗，数据库设计及前后端实现
    • 等等.......
    1. 项目优化：
    • ui优化：封装了原框架使用的card组件，让原来的card组件看起来更加丰富。
    • 性能优化：通过dynamic + <script strategy="afterInteractive"><script>实现组件懒加载

    1. 项目整体包含一个管理端、一个用户端、一个移动端。
    • 管理端是由本人从0到1开始搭建的后台系统，整体的框架为vite+vue3+elementPlus。项目地址为：管理后台 (opens new window)
    • 用户端则是公司已有的系统进行重构、ui改版、增加新功能点。项目地址为:地名地址信息系统 (opens new window)
    • 移动端是基于公司原有的的hybirdApp进行开发。
